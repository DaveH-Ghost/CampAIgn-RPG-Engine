# Version 1.0.0 Changelog (Realm-Fabric engine)

**Purpose:** Library-first **1.0** release — single `realm_fabric` package on PyPI, no bundled CLI, no `Session.run_command()`.

**Baseline:** **0.7.2** (`v0.7.2`). Pre-1.0 work preserved on branch `archive/0.7.x`.

**Status:** ✅ **Implemented** — tag **`v1.0.0`** pending PyPI publish.

See also: [MIGRATION-0.7-to-1.0.md](../MIGRATION-0.7-to-1.0.md), [v0.7.2-changelog.md](v0.7.2-changelog.md).

---

## Breaking changes

### Package layout

- Collapsed `src/` into a single **`realm_fabric/`** package at repo root (flat layout).
- Import only `from realm_fabric import …` — no `src.*` paths.

### CLI removed

- Deleted `realm` console entry point (`main.py`, stepper, CLI docs).
- Manual GM testing: use [Realm-Studio](https://github.com/DaveH-Ghost/Realm-Studio) (separate GitHub repo).

### String command layer removed

- `Session.run_command()` and private `_cmd_*` handlers removed.
- `CommandResult` removed from public API.
- Area string parsers moved to `realm_fabric.area_edit` (for app-owned dispatch, not public `__all__`).

### Examples trimmed

- Removed `examples/minimal-server/`, `examples/reference_handlers/`, `examples/custom_memory/`.
- Kept `examples/lorebook/`; HTTP/reference handlers live in Realm-Studio.

### Realm-Studio external

- GM web app moved to [github.com/DaveH-Ghost/Realm-Studio](https://github.com/DaveH-Ghost/Realm-Studio).
- Studio depends on `realm-fabric>=1.0.0` from PyPI (or local editable during co-dev).

---

## Additions / fixes

- Typed `Session` world API is the only editing surface (`create_object`, `edit_agent`, `create_area`, …).
- Public API surface tests and packaging tests updated for 1.0 layout.
- Migration guide: [MIGRATION-0.7-to-1.0.md](../MIGRATION-0.7-to-1.0.md).
- **541** engine tests pass; Studio wheel smoke test passes on `realm_fabric-1.0.0` wheel.

---

## Release

1. Tag **`v1.0.0`** and push to GitHub.
2. `uv build` → `uv publish` to PyPI.
3. Bump [Realm-Studio](https://github.com/DaveH-Ghost/Realm-Studio) to PyPI-only `realm-fabric>=1.0.0` (remove `[tool.uv.sources]` path override).

Save format **`snapshot_version`** remains **4** — no bump required for 1.0.
