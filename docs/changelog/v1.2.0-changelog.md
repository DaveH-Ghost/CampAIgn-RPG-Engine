# Version 1.2.0 Changelog (CampAIgn-RPG-Engine)

**Purpose:** Plugin primitives for apps and CampAIgn-RPG-Studio — session extensions, events, turn verbs, and prompt slots.

**Baseline:** **1.0.0** / **1.1.x** (Studio player turns shipped separately).

**Status:** ✅ **Implemented** — tag **`v1.2.0`** pending PyPI publish.

See also: [plugins.md](../guides/plugins.md), [v1.0.0-changelog.md](v1.0.0-changelog.md).

---

## Additions

### Session extensions

- `session.extensions` dict round-trips in saves and snapshots (`extensions` field).
- `Session.get_extension(plugin_id)` / `Session.set_extension(plugin_id, value)` helpers.

### Event registry

- `register_event_listener(event, fn, *, plugin_id=...)`
- `unregister_event_listeners(plugin_id)`
- `emit_session_event(session, event, **payload)` (apps) and internal `_emit_event` from Session mutators.
- Stable v1 events: `turn_committed`, `agent_moved`, `object_created` / `object_removed` / `object_edited`, `agent_created` / `agent_removed` / `agent_edited`.

### Turn verbs

- `register_turn_verb(verb_id, executor, *, description=..., validate_turn=...)`
- `list_registered_turn_verbs()`, `run_turn_verb(...)` — compound turns with `action: "verb"`.
- Simulation and witness integration for verb outcomes.

### Prompt slots

- `register_prompt_slot(name, renderer, *, description=...)`
- `PromptBlock(type="plugin_slot", name="...")` in prompt layout validation and rendering.
- `plugin_slot` block type for Studio prompt editor.

### World editing

- `Session.create_object(..., object_id=...)` — optional stable id when placing objects (used by inventory drop).

---

## Compatibility

- **`snapshot_version`** remains **4** — no save-format bump.
- **No breaking API removals** vs 1.0.x; additive release.
- Apps on `campaign-rpg-engine>=1.0.0` can upgrade to 1.2.0 without migration beyond registering new plugin code at startup.

---

## Studio (separate repo)

CampAIgn-RPG-Studio **1.2.0** adds the plugin host, Plugins tab, and reference **inventory** plugin. Requires **`campaign-rpg-engine>=1.2.0`**.

---

## Release

1. Tag **`v1.2.0`** and push to GitHub.
2. `uv build` → `uv publish` to PyPI.
3. Publish CampAIgn-RPG-Studio **1.2.0** (depends on PyPI `campaign-rpg-engine>=1.2.0`).
