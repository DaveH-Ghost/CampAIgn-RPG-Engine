# Version 1.4.0 Changelog (CampAIgn-RPG-Engine)

**Purpose:** Turn-verb pathing and witness exclusions — primitives for agent-to-agent inventory actions in Studio.

**Baseline:** **1.3.1**

**Status:** Implemented — tag **`v1.4.0`** pending PyPI publish.

---

## Additions

### Turn verb pathing (opt-in)

Plugins can register ranged turn verbs that path toward another agent before execution (same move-budget rules as interact pathing):

- `register_turn_verb(..., path_range=..., path_target_from_turn=...)`
- `run_turn_verb_phases`, `verb_turn_has_pathing`
- Compound turns with a pathing verb **ignore** explicit `move` for that turn (like `action: "interact"`).
- Records a **move** step when pathing changes the actor's tile; fails with a too-far message if still out of range after the budget.

### Witness exclusions

- `ActionOutcome.passive_witness_exclude_agent_ids` — observer agent ids that should not receive a step's passive witness broadcast.
- `TurnStep.passive_witness_exclude_agent_ids` — round-trips in memory module serialization when non-empty.

Useful when a verb emits a **private** `emit_area_event` to one agent and a **public** passive line for everyone else (e.g. inventory `show`).

### Perception

- `nearest_standable_in_chebyshev_range(agent, area, target_pos, action_range)` — closest enterable tile within Chebyshev range of a grid position.

## Compatibility

- **`snapshot_version`** remains **5** — no save-format bump.
- Additive API only; apps on `>=1.3.1` can upgrade without migration beyond registering updated plugin code.
