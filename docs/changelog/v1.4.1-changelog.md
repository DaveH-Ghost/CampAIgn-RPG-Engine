# Version 1.4.1 Changelog (CampAIgn-RPG-Engine)

**Purpose:** Interaction handlers may return a full ``ActionOutcome`` so plugins can supply fail ``result`` / ``passive_result`` without mutating action templates (skills checks).

**Baseline:** **1.4.0**

**Status:** Implemented — GitHub tag **`v1.4.1`**; PyPI deferred until 1.5.0.

---

## Additions

### Handler ``ActionOutcome`` return (1.4.1)

``InteractionHandler`` / ``run_interaction_handler`` may return:

| Return | Interact behavior |
|--------|-------------------|
| ``None`` | Success — format ``action.result`` / ``action.passive_result`` (unchanged) |
| ``str`` | Abort — actor-only message, empty passive (unchanged) |
| ``ActionOutcome`` | **Final** interact outcome — skip success templates |

Use this when a handler needs different success vs fail witness text (e.g. Studio skills plugin ``skill_check``).

Triggers still invoke handlers for side effects; an ``ActionOutcome`` return from a trigger handler is ignored for area-event text (trigger messaging stays on ``passive_result``).

### Generic handler params on object actions (1.4.1)

``create-object`` / ``edit-object ... add-action`` accept **extra** ``key value`` pairs beyond the reserved action fields. Those pairs become ``handler_params`` and are validated by the handler's ``validate_params`` (e.g. ``stat STR dc 12`` for Studio ``skill_check``). ``dest-area`` / ``dest-at`` for ``move_area`` work the same way.

---

## Compatibility

- **``snapshot_version``** remains **5**.
- Existing handlers returning ``None`` or ``str`` are unchanged.
