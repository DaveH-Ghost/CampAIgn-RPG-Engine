# Version 1.7.1 Changelog (CampAIgn-RPG-Engine)

**Purpose:** Joint release with Studio **1.7.1** — more resilient compound-turn parsing and intentional interact positioning.

**Baseline:** **1.6.1**

**Status:** Implemented — tag **`v1.7.1`** on publish.

---

## Changes

### Compound-turn JSON repair

- `normalize_compound_turn_payload` strips a leading `.` from keys (e.g. `.reasoning` → `reasoning`).
- Some models (notably Featherless/DeepSeek-style structured output) emit dotted field names that previously failed as `ERR:INVALID_JSON`.
- When both dotted and plain keys are present, the plain key wins.

### Interact pathing vs explicit `move`

- When `action` is `interact`, an explicit compound `move` is no longer always ignored.
- If that move would leave the agent **in interact range** this turn, nav honors it (no overwrite to nearest in-range tile).
- If the move would leave them out of range (or is omitted), interact still auto-paths as before.
- Supports intentional positioning for future combat/approach play.

---

## App developer impact

- CampAIgn-RPG-Studio **1.7.1** requires **`campaign-rpg-engine>=1.7.1`**.
- No snapshot format change (`snapshot_version` remains **5**).
