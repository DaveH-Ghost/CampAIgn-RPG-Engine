# Version 1.7.4 Changelog (CampAIgn-RPG-Engine)

**Purpose:** Joint release with Studio **1.7.4** — keep default-prompt token regression tests in sync with current compound rules, and harden a flaky LLM-failure unit test.

**Baseline:** **1.7.3**

**Status:** Implemented — tag **`v1.7.4`** on publish.

## Changes

### Prompt token budgets

- `DEFAULT_PROMPT_TOKEN_BUDGET` raised **650 → 750** so the empty demo prompt stays under the regression cap after `[far]` / emote / verb-line growth in `compound_rules`.
- `compound_rules` section budget in tests raised **200 → 350**.

### Tests

- `test_llm_failure_does_not_increment_session_turn` now patches `campaign_rpg_engine.get_compound_turn` via the package module (a local import was bypassing the monkeypatch and hitting a live LLM).

## Compatibility

- CampAIgn-RPG-Studio **1.7.4** requires **`campaign-rpg-engine>=1.7.4`**.
- No snapshot format change (`snapshot_version` remains **5**).
- No public API change vs **1.7.3**.
